# Credit Risk Explainer

A local, containerized ML application that predicts credit default risk for loan applicants and provides transparent, human-readable explanations via SHAP values.

## Overview

The system is composed of three components:

- **Trainer** (`train.py`) — preprocesses the dataset, trains an XGBoost classifier inside a Scikit-learn Pipeline, and saves the model artifact to disk.
- **API** (`api/`) — a FastAPI service that loads the trained pipeline and serves predictions over HTTP.
- **UI** (`ui/`) — a Streamlit web app where users enter loan application details and view the prediction alongside a SHAP Waterfall Plot explaining which features drove the result.

Predictions are labeled **High Risk** or **Low Risk** based on a probability threshold of 0.5, with color-coded output and per-feature SHAP attributions rendered in the browser.

## Architecture

```
train.py  ──writes──►  model/pipeline.joblib
                              │
                         volume mount
                              │
Browser ──:8501──► [ui service] ──POST /predict──► [api service :8000]
```

The UI and API run as Docker containers on the same Compose network. The UI communicates with the API using the internal service name `http://api:8000`.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and [Docker Compose](https://docs.docker.com/compose/install/)
- Python 3.10+ (only needed for the training step, which runs on the host)

## Quickstart

### 1. Install training dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the model

Run the training script from the project root. It reads `data/credit_risk_dataset.csv`, fits the pipeline, and writes the artifact to `model/pipeline.joblib`.

```bash
python train.py
```

Expected output:
```
Validation ROC-AUC: 0.9xxx
Pipeline saved to 'model/pipeline.joblib'
```

### 3. Start the application

```bash
docker-compose up --build
```

Docker Compose will build both service images and start the containers. The UI waits for the API health check to pass before becoming available.

### 4. Open the UI

Navigate to [http://localhost:8501](http://localhost:8501) in your browser.

Fill in the loan application details in the sidebar and click **Predict** to see the risk assessment and SHAP explanation.

## Input Features

| Feature | Description | Type |
|---|---|---|
| Age | Applicant age | Integer (18–100) |
| Annual Income | Gross annual income in USD | Integer (≥ 1) |
| Home Ownership | `RENT`, `OWN`, `MORTGAGE`, or `OTHER` | String |
| Employment Length | Years employed (optional) | Float (≥ 0) |
| Loan Intent | Purpose of the loan | String (6 options) |
| Loan Grade | Credit grade assigned to the loan | String (A–G) |
| Loan Amount | Requested loan amount in USD | Integer (≥ 1) |
| Interest Rate | Annual interest rate in % (optional) | Float (≥ 0) |
| Loan % of Income | Loan amount as a fraction of income | Float (0.0–1.0) |
| Previous Default | Whether applicant has a prior default on file | `Y` or `N` |
| Credit History Length | Years of credit history | Integer (≥ 0) |

## API Endpoints

The API is also accessible directly at `http://localhost:8000`.

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Liveness check — returns `{"status": "ok"}` |
| `POST` | `/predict` | Accepts `LoanFeatures` JSON, returns prediction + SHAP values |

Example request:
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "person_age": 30,
    "person_income": 50000,
    "person_home_ownership": "RENT",
    "person_emp_length": 5.0,
    "loan_intent": "PERSONAL",
    "loan_grade": "B",
    "loan_amnt": 10000,
    "loan_int_rate": 10.5,
    "loan_percent_income": 0.2,
    "cb_person_default_on_file": "N",
    "cb_person_cred_hist_length": 3
  }'
```

## Running Tests

Tests require the trained model artifact to be present. Run from the project root:

```bash
pytest tests/
```
